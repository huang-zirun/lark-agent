# 定位与修复计划：test_generation_and_execution 阶段失败

## 问题现象

错误信息：`Stage test_generation_and_execution failed: OpenAI API request failed: Agent test_agent execution failed`

## 根因分析

### 原因 1：test_agent 的 output_schema 与实际期望输出不匹配

- `profiles.py` 中 `test_agent.output_schema = "TestAgentOutput"`
- `artifacts.py` 中 `"test" -> "test_report"`
- 因此 runner 要求 LLM 按 `test_report` schema 输出
- 但 `system_prompt` 要求输出 `change_set`（测试文件变更集）
- LLM 被同时要求输出两种格式，导致输出混乱或校验失败

### 原因 2：_execute_test_stage 与 run_agent 的 schema 校验冲突

- `_execute_test_stage` 期望 `run_agent` 返回 `test_change_set` 或 `change_set`
- 但 `run_agent` 的 `_validate_and_fix_output` 会过滤掉所有不在 `ARTIFACT_TYPE_TO_SCHEMA` 中的 key
- `test_change_set` 不在 `ARTIFACT_TYPE_TO_SCHEMA` 中，会被过滤
- 如果 LLM 返回 `change_set`，虽然 key 存在，但 schema 是 `ChangeSet`（文件变更集），而 prompt 要求的是测试代码变更集，逻辑上可行但 profile 配置错误导致 schema 要求变成了 `test_report`

### 原因 3：Mock Agent 与真实 Agent 行为不一致

- `mock_test_agent` 直接返回 `test_report`，跳过了测试代码生成和测试执行
- 真实流程中 `_execute_test_stage` 需要先得到测试代码 `change_set`，再 apply、再执行测试命令
- 这意味着 Mock 流程没有覆盖真实 LLM 路径的问题

### 原因 4：OpenAICompatibleProvider 缺乏结构化输出约束

- 当前仅通过 prompt 拼接 schema 说明，未使用 `response_format={"type": "json_object"}`
- 导致 LLM 可能返回非 JSON 或不符合 schema 的内容

## 修复步骤

### Step 1：修正 test_agent 的 profile，使其输出 change_set

文件：`backend/app/agents/profiles.py`

- 将 `test_agent.output_schema` 从 `"TestAgentOutput"` 改为 `"CodePatchAgentOutput"`（或新增 `"TestChangeSetOutput"` 映射到 `"change_set"`）
- 修改 `system_prompt`，明确要求输出 `change_set` 格式的测试文件

### Step 2：修改 artifacts.py 的映射关系

文件：`backend/app/schemas/artifacts.py`

- 在 `OUTPUT_SCHEMA_TO_ARTIFACT_TYPE` 中新增 `"testchange": "change_set"` 映射
- 或直接将 `"test"` 的映射改为 `"change_set"`（但需确认其他代码是否依赖 `"test" -> "test_report"`）
- 更安全的做法：新增 `"testchange": "change_set"`，然后 profile 使用 `"TestChangeAgentOutput"`

### Step 3：修改 _execute_test_stage 的错误处理与回退逻辑

文件：`backend/app/core/execution/stage_runner.py`

- 在 `run_agent` 调用失败时，增加更详细的日志
- 确保 `test_change_set` 提取逻辑健壮（已存在 `or` 回退）
- 在测试代码 apply 失败时，不应直接失败，而应记录失败并生成包含错误信息的 test_report

### Step 4：增强 OpenAICompatibleProvider 的 JSON 可靠性

文件：`backend/app/core/provider/openai_compatible.py`

- 当 `schema` 存在时，在 payload 中添加 `response_format={"type": "json_object"}`
- 这要求 `model` 支持 JSON mode（gpt-4o 支持）

### Step 5：验证修复

- 确认 `run_agent("test_agent", ...)` 返回的 result 包含 `change_set` key
- 确认 `_execute_test_stage` 能正确提取测试文件、apply、执行测试命令
- 确认最终返回的 `test_report` 符合 schema

## 验收标准

- [ ] test_agent 的 profile 配置与期望输出一致
- [ ] run_agent 调用 test_agent 时不再因 schema 不匹配而失败
- [ ] _execute_test_stage 能完整执行：生成测试代码 -> apply -> 执行测试 -> 生成 test_report
- [ ] OpenAICompatibleProvider 在 schema 模式下使用 json_object response_format
- [ ] Mock Agent 流程仍然正常
