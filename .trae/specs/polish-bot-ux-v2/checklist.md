# Checklist

## 启动就绪消息保障
- [x] `default_chat_id` 已配置时，`devflow start` 启动后向该聊天发送欢迎卡片
- [x] `default_chat_id` 未配置时，控制台输出明确引导信息（机器人已就绪、可在飞书中发消息、配置方法）
- [x] `default_chat_id` 未配置时，用户首次发消息的确认回复附带简短使用指引
- [x] 单元测试覆盖：未配置时控制台输出、首次交互附带指引

## 待澄清问题交互机制
- [x] 需求分析完成后，`ready_for_next_stage=false` 且 `open_questions` 非空时暂停流水线
- [x] 暂停时写入 `stage=clarification`、`status=waiting_clarification` 的检查点
- [x] 暂停时发送待澄清问题卡片（橙色头部、逐条列出问题、操作指引）
- [x] 用户回复非跳过命令时，回答追加到 open_questions 对应条目的 answer 字段
- [x] 用户回复后重新评估质量信号并继续方案设计
- [x] 用户回复 `继续`/`skip`/`跳过` 时标记所有未回答问题为 skipped，继续方案设计
- [x] 跳过澄清时检查点记录 quality_at_approval 包含跳过警告
- [x] `ready_for_next_stage=true` 或 `open_questions` 为空时不暂停，正常流转
- [x] 消息路由优先级正确：系统命令 > 检查点命令 > 拒绝理由 > 待澄清问题回复 > 仓库恢复 > 合并窗口判断 > 新需求
- [x] 单元测试覆盖：暂停与卡片、用户回复继续、跳过继续、无问题正常流转

## 中间产物文档发布为飞书云文档

- [x] 方案评审卡片包含"[查看完整方案文档]({url})"飞书链接
- [x] 代码评审卡片包含"[查看完整评审报告]({url})"飞书链接
- [x] PRD 预览卡片顶部包含"[查看完整 PRD 文档]({url})"链接
- [x] 方案评审卡片文件变更预览增加到最多 10 项
- [x] 代码评审卡片问题预览增加到最多 10 项
- [x] 审批命令使用代码块包裹，便于一键复制
- [x] 文档发布失败时卡片显示本地路径、run.json 记录错误
- [x] 单元测试覆盖：发布成功时卡片包含链接、发布失败时降级显示
