# Checklist

## 消息缓冲与合并
- [x] `MessageBuffer` 类按 `chat_id + sender_id` 分组缓冲消息，合并窗口内消息正确拼接
- [x] `config.json` 支持 `message_merge_window_seconds` 配置项，默认 5 秒
- [x] `run_start_loop` 事件流经过 `MessageBuffer` 缓冲后再进入 `process_bot_event`
- [x] 缓冲窗口内收到后续消息时回复"已追加到当前需求中"
- [x] 系统命令和检查点命令绕过合并窗口，立即传递
- [x] 单元测试覆盖：窗口内合并、窗口外独立、多用户隔离

## 消息队列
- [x] `UserMessageQueue` 类按 `chat_id + sender_id` 分组管理消息队列
- [x] `config.json` 支持 `max_queue_size` 配置项，默认 5
- [x] 同一用户消息串行处理，不同用户消息可并行处理
- [x] 队列满时丢弃最早消息并回复提示
- [x] 单元测试覆盖：串行保证、并行隔离、队列溢出

## 即时确认回复
- [x] 收到需求消息后立即发送确认回复（含运行 ID）
- [x] 收到检查点命令后立即发送确认回复
- [x] 确认回复使用幂等键防止重复
- [x] 确认发送失败不阻塞后续处理
- [x] 单元测试验证确认回复发送和幂等键

## 阶段状态通知
- [x] `config.json` 支持 `progress_notifications_enabled` 配置项（`interaction.progress_notifications_enabled`），默认 true
- [x] 阶段中文名映射表完整覆盖所有 6 个阶段
- [x] 阶段开始时发送"📋 {阶段中文名} 进行中… （{已完成数}/{总数}）"
- [x] 阶段完成时发送"✅ {阶段中文名} 已完成"
- [x] 阶段失败时发送"❌ {阶段中文名} 失败：{错误摘要}"
- [x] 状态通知使用幂等键
- [x] `progress_notifications_enabled = false` 时不发送阶段通知（仅保留确认和最终结果）
- [x] 单元测试覆盖：启用/禁用、格式正确、幂等

## 欢迎消息
- [x] `config.json` 支持 `default_chat_id` 配置项
- [x] `devflow start` 启动后发送欢迎卡片
- [x] 欢迎卡片包含：标题、输入示例、命令列表、配置摘要
- [x] 未配置 `default_chat_id` 时跳过发送并记录日志
- [x] 单元测试覆盖：内容正确、跳过逻辑

## `/help` 和 `/status` 命令
- [x] 命令解析支持 `/help`、`/帮助`、`/status`、`/状态`
- [x] `/help` 回复使用指引卡片，不创建运行
- [x] `/status` 查找活跃运行并回复状态摘要，无活跃运行时回复提示
- [x] 消息路由优先级：系统命令 > 检查点命令 > 拒绝理由 > 仓库恢复 > 新需求
- [x] 系统命令和检查点命令绕过消息合并窗口
- [x] 单元测试覆盖：命令解析和处理逻辑

## 集成验证
- [x] 多消息合并场景：5 秒内发送 3 条消息，合并为 1 个事件（单元测试验证）
- [x] 消息队列场景：同一用户串行处理，不同用户并行（单元测试验证）
- [x] 即时确认：需求消息和检查点命令均发送确认回复（单元测试验证）
- [x] 阶段通知：6 个阶段的 started/completed/failed 通知格式正确（单元测试验证）
- [x] 欢迎消息：启动时发送欢迎卡片（单元测试验证）
- [x] `/help` 和 `/status` 命令正常工作（单元测试验证）
