# 审批交互问题分析与实施日志

## 日期
2026-05-03

## 问题
用户收到技术方案评审卡片后，无法通过按钮、审批或拒绝并附带理由来回复。

## 根本原因
1. 卡片中的"同意/拒绝"是纯文本提示，不是可点击按钮
2. `lark-cli` 1.0.23 不支持卡片按钮回调事件
3. 用户需要在聊天窗口手动输入 `Approve {run_id}` / `Reject {run_id}` 命令

## 方案演进

### 第一阶段：原生审批（Native Approval）
- 使用 `POST /open-apis/approval/v4/instances` 创建原生审批实例
- **问题**：需要预先在飞书管理后台创建审批模板，个人用户无权限
- `config.json` 中 `approval.definition_code` 为空时，审批逻辑被跳过，回退到卡片模式

### 第二阶段：三方审批（External Approval）— 当前方案
- 使用 `POST /open-apis/approval/v4/external_approvals` 自动创建三方审批定义
- **优势**：无需后台模板，程序自动创建，个人用户可用
- 审批实例通过 `POST /open-apis/approval/v4/external_instances` 同步到飞书
- 用户在飞书审批应用中看到待办，点击"同意"或"拒绝"（拒绝需填理由）

## 配置方式
```json
{
  "approval": {
    "enabled": true,
    "definition_code": "",
    "poll_interval_seconds": 60
  }
}
```
- `definition_code` 留空时，系统自动创建三方审批定义
- 填入已有值时，尝试复用该定义
